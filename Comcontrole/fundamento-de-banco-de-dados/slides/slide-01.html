Dado, informação e conhecimento — os três pilares
Do dado bruto ao conhecimento aplicado

Antes de falar de tabelas, SQL ou modelagem, é preciso entender uma distinção que sustenta toda a área de Banco de Dados: dado, informação e conhecimento não são sinônimos — são estágios diferentes de um mesmo processo de transformação.

Conhecimento
Informação combinada com experiência, contexto e julgamento — permite decisão e ação
Informação
Dado organizado, processado e contextualizado — já tem significado
Dado
Fato bruto, sem contexto — um número, uma palavra, um registro isolado
Por que essa distinção importa para Banco de Dados? Um SGBD armazena e organiza dados. Mas o objetivo final de qualquer sistema de informação é produzir informação útil — e, na mão certa, essa informação vira conhecimento que orienta decisões. Modelar um banco de dados bem é, em essência, projetar a ponte entre o dado bruto e a informação que alguém vai usar.
Exemplos do cotidiano
EstágioExemploPor quê
Dado27,3Um número isolado, sem unidade nem contexto — pode ser qualquer coisa
InformaçãoA temperatura em Teresina às 14h é 27,3°CO dado ganhou contexto (o quê, onde, quando) e virou algo compreensível
ConhecimentoEm dias acima de 27°C às 14h, é melhor regar as plantas só à noiteA informação foi cruzada com experiência prévia e virou uma orientação prática

Note que o mesmo padrão se repete em qualquer contexto: uma nota isolada (dado) → o boletim do aluno (informação) → a decisão do professor de reforçar um conteúdo específico (conhecimento aplicado).

Aula 2 · ~50 minutos
O profissional de Banco de Dados no contexto organizacional
Quem trabalha com dados dentro de uma empresa

Não existe um único "profissional de banco de dados" — existem vários papéis complementares, cada um responsável por uma parte do ciclo de vida do dado, da coleta até a decisão final. Conhecer essas funções ajuda você a entender onde a formação em IA se encaixa nesse ecossistema.

PapelFoco principalExemplo de tarefa
DBA (Administrador de BD)Manter o SGBD funcionando: performance, backup, segurança, disponibilidadeConfigurar rotina de backup automático e monitorar uso de espaço em disco
DesenvolvedorConstruir sistemas que leem e gravam dados no bancoEscrever o código que salva um novo pedido de compra no banco
Analista de dadosExtrair informação e insights a partir dos dados existentesCriar um relatório de vendas por região usando consultas SQL
Engenheiro de dadosConstruir a infraestrutura que move e transforma dados em escalaMontar um pipeline que carrega dados de vendas todo dia às 6h
Onde a IA entra nisso? Um engenheiro/cientista de IA depende diretamente da qualidade do trabalho desses profissionais — um modelo de machine learning é tão bom quanto os dados que o alimentam. Entender bancos de dados não é "matéria de outro curso": é a base que sustenta qualquer projeto sério de IA.
Onde esses profissionais atuam
  • Times de TI/Infraestrutura: DBAs cuidando da saúde dos servidores de banco de dados.
  • Times de produto/desenvolvimento: desenvolvedores integrando aplicações aos bancos de dados do dia a dia.
  • Times de dados/BI: analistas e engenheiros de dados transformando dados brutos em relatórios e pipelines.
  • Times de IA/ML: cientistas de dados consumindo dados já organizados para treinar modelos.
Exercício 02 — Associando função e responsabilidade

Associe cada situação abaixo ao profissional mais responsável por resolvê-la (DBA, Desenvolvedor, Analista de dados ou Engenheiro de dados):

(a) O sistema de vendas está lento porque o banco de dados está sobrecarregado às sextas-feiras à tarde
(b) A diretoria pediu um relatório comparando vendas do primeiro trimestre com o mesmo período do ano passado
(c) É preciso criar a tela de cadastro de clientes que grava os dados no banco
(d) Os dados de vendas de 5 lojas diferentes precisam ser consolidados automaticamente todas as noites em um único repositório
