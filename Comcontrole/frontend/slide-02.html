Componentes funcionais e JSX — o modelo mental do React
O que é React, de verdade?

React é uma biblioteca JavaScript para construir interfaces. A ideia central é simples: a UI é uma função do estado. Você descreve como a tela deve parecer dado um estado, e o React garante que o DOM reflita isso.

Estadodados / variáveis
→
Componentefunção JS
→
JSXdescreve a UI
→
DOMo que o usuário vê
Mudou o estado → o React atualiza o DOM automaticamente. Você nunca escreve document.getElementById em React. Você descreve a UI e o React faz o resto.
Componente funcional — a unidade básica

Um componente React é uma função JavaScript que começa com letra maiúscula e retorna JSX. É só isso.

JSX — primeiro componente
// Componente mais simples possível
function OlaMundo() {
  return <h1>Olá, React!</h1>;
}

// Exportar para usar em outros arquivos
export default function CartaoUsuario() {
  const nome = "Ana Lima";
  const papel = "Engenheira de ML";

  return (
    <div className="cartao">          {/* className, não class! */}
      <h2>{nome}</h2>                   {/* { } para JS dentro do JSX */}
      <p>{papel}</p>
      <p>Hoje: {new Date().toLocaleDateString('pt-BR')}</p>
    </div>
  );
}

// Usar o componente em outro lugar
function App() {
  return (
    <main>
      <CartaoUsuario />   {/* componente como tag HTML */}
      <CartaoUsuario />   {/* reutilizável! */}
    </main>
  );
}
JSX não é HTML. Diferenças importantes: className em vez de class · htmlFor em vez de for · toda tag deve fechar (<img />) · expressões JS ficam dentro de { } · retornar múltiplos elementos exige um elemento pai ou <>...</> (Fragment).
Composição — construindo com peças

React encoraja dividir a UI em componentes pequenos e compô-los. Um componente pode conter outros componentes — essa é a estrutura de qualquer app real.

JSX — composição de componentes
// Componentes pequenos e focados
function Avatar() {
  return <div className="avatar">🤖</div>;
}

function NomeBadge() {
  return <span className="badge">IA Assistant</span>;
}

function ChatHeader() {
  // Compõe Avatar + NomeBadge
  return (
    <header className="chat-header">
      <Avatar />
      <NomeBadge />
    </header>

// Fragment — retornar sem div extra
function DoisItens() {
  return (
    <>
      <p>Primeiro</p>
      <p>Segundo</p>
    </>
  );
}

Props e useState — dados que fluem e dados que mudam
Props — dados de pai para filho

Props são os parâmetros dos componentes. O componente pai passa dados, o filho os recebe como objeto. O fluxo é sempre de cima para baixo — nunca de filho para pai via props.

<App /> — componente pai
↓ passa props
<CartaoMensagem autor="Ana" texto="Olá!" />
props.autor → "Ana"
props.texto → "Olá!"
JSX — Props na prática
// Receber props via destructuring (padrão de mercado)
function MensagemChat({ autor, texto, horario, isBot = false }) {
  return (
    <div className={isBot ? "msg msg--bot" : "msg msg--user"}>
      <span className="msg-autor">{autor}</span>
      <p className="msg-texto">{texto}</p>
      <time>{horario}</time>
    </div>
  );
}

// Usar o componente — passando props como atributos
function App() {
  return (
    <section>
      <MensagemChat
        autor="Ana"
        texto="Qual modelo devo usar para RAG?"
        horario="14:32"
      />
      <MensagemChat
        autor="IA"
        texto="Para RAG, recomendo text-embedding-3-small..."
        horario="14:32"
        isBot={true}
      />
    </section>
  );
}
Props são imutáveis dentro do componente filho. Um filho nunca modifica as props que recebe. Se precisar comunicar algo ao pai, o pai passa uma função como prop — isso é o padrão de "lifting state up" que veremos na Semana 3.
useState — estado reativo

Estado é a memória do componente. Quando o estado muda, o React re-renderiza o componente automaticamente. useState retorna um array com dois elementos: o valor atual e a função para atualizá-lo.

estado inicialuseState(valorInicial)

JSX — useState na prática
import { useState } from 'react';

export default function ContadorMensagens() {
  // [valor atual, função de atualização] = useState(valor inicial)
  const [contador, setContador] = useState(0);
  const [nome, setNome]         = useState('');
  const [ativo, setAtivo]       = useState(false);

  const incrementar = () => setContador(c => c + 1); // forma funcional (segura)

  return (
    <div>
      <p>Mensagens enviadas: {contador}</p>

      <input
        value={nome}
        onChange={e => setNome(e.target.value)}  /* controlado */
        placeholder="Seu nome"
      />

      <button onClick={incrementar}>
        Enviar mensagem
      </button>

      <p>Olá, {nome || 'visitante'}! Você enviou {contador} msgs.</p>
    </div>
  );
}
NUNCA modifique o estado diretamente. contador = contador + 1 não funciona — React não vai detectar a mudança. Sempre use a função setter: setContador(c => c + 1). Para objetos e arrays, crie uma cópia com spread: setLista([...lista, novoItem]).
useState com objetos e arrays
JSX — estado com arrays (base do chat)
import { useState } from 'react';

export default function ChatSimples() {
  const [mensagens, setMensagens] = useState([
    { id: 1, texto: 'Olá! Como posso ajudar?', isBot: true }
  ]);
  const [input, setInput] = useState('');

  const enviarMensagem = () => {
    if (!input.trim()) return;

    // ✅ Adicionar ao array: spread + novo item
    setMensagens(prev => [...prev, {
      id: Date.now(),
      texto: input,
      isBot: false
    }]);

    setInput(''); // limpa o campo

    // Resposta automática do bot (estática por enquanto)
    setTimeout(() => {
      setMensagens(prev => [...prev, {
        id: Date.now() + 1,
        texto: 'Entendido! Na Semana 5 vou ser uma IA de verdade. 🤖',
        isBot: true
      }]);
    }, 800);
  };

  return (
    <div className="chat">
      {/* renderizar lista — aula 3 */}
      <input value={input} onChange={e => setInput(e.target.value)} />
      <button onClick={enviarMensagem}>Enviar</button>
    </div>
  );
}

Renderização condicional, listas e o componente de chat completo
Renderização condicional — mostrar ou esconder

Em React você não usa if fora do return para controlar o que aparece na tela. Existem três padrões que você vai usar todo dia:

JSX — padrões de condicional
function PainelIA({ carregando, erro, resposta, logado }) {
  return (
    <div>

      {/* Padrão 1: operador ternário — if/else */}
      {carregando
        ? <p>Gerando resposta...</p>
        : <p>Pronto!</p>
      }

      {/* Padrão 2: && — só renderiza se verdadeiro */}
      {erro && <div className="erro">Erro: {erro}</div>}

      {resposta && <div className="resposta">{resposta}</div>}

      {/* Padrão 3: retorno antecipado (early return) — fora do JSX */}
      {/* if (!logado) return <Redirect to="/login" />; — antes do return */}

      {/* Padrão 4: nulo não renderiza nada */}
      {logado ? <BotaoLogout /> : null}

    </div>
  );
}
Armadilha com &&: {"{"}0 && <Componente />{"}"} vai renderizar o número 0 na tela, não nada. Use {"{"}lista.length > 0 && ...{"}"} ou {"{"}!!valor && ...{"}"} para garantir um booleano.
Renderizando listas com map — e o atributo key

Para renderizar arrays em React, você usa .map(). Cada item obrigatoriamente precisa de um key único — sem ele, React não consegue identificar quais itens mudaram.

JSX — listas com map e key
const mensagens = [
  { id: 1, texto: 'Olá!',           isBot: true  },
  { id: 2, texto: 'Como vai?',       isBot: false },
  { id: 3, texto: 'Tudo bem, e você?', isBot: true  },
];

function ListaMensagens({ mensagens }) {
  return (
    <div className="lista">
      {mensagens.map(msg => (
        <MensagemChat
          key={msg.id}            {/* key obrigatório e único */}
          texto={msg.texto}
          isBot={msg.isBot}
        />
      ))}

      {/* Lista vazia — condicional */}
      {mensagens.length === 0 && (
        <p className="vazio">Nenhuma mensagem ainda.</p>
      )}
    </div>
  );
}

// ❌ Nunca use o índice como key em listas dinâmicas
// mensagens.map((msg, index) => <Item key={index} />) — EVITAR
// ✅ Use um ID único do dado
Por que key importa? React usa o key para identificar qual item foi adicionado, removido ou reordenado. Sem key único, o React pode reutilizar o DOM errado — causando bugs visuais difíceis de detectar.

JSX — ChatBot.jsx completo
import { useState, useRef, useEffect } from 'react';

// Respostas estáticas do bot (Semana 5 vai substituir por API real)
const RESPOSTAS_BOT = [
  'Interessante! Me conte mais sobre isso.',
  'Ótima pergunta. Na Semana 5 vou usar a OpenAI para responder melhor!',
  'Entendido! Estou processando sua mensagem... 🤔',
  'Isso está relacionado a IA? Adoro esse assunto!',
];

export default function ChatBot() {
  const [mensagens, setMensagens] = useState([
    { id: 1, texto: 'Olá! Como posso ajudar?', isBot: true }
  ]);
  const [input, setInput]         = useState('');
  const [digitando, setDigitando] = useState(false);
  const fimRef                    = useRef(null);

  // Scroll automático para o final
  useEffect(() => {
    fimRef.current?.scrollIntoView({ behavior: 'smooth' });
  }, [mensagens]);

  const enviar = () => {
    if (!input.trim() || digitando) return;

    // Adiciona mensagem do usuário
    setMensagens(prev => [...prev, {
      id: Date.now(), texto: input, isBot: false
    }]);
    setInput('');
    setDigitando(true);

    // Simula bot "digitando" e responde
    setTimeout(() => {
      const resposta = RESPOSTAS_BOT[
        Math.floor(Math.random() * RESPOSTAS_BOT.length)
      ];
      setMensagens(prev => [...prev, {
        id: Date.now(), texto: resposta, isBot: true
      }]);
      setDigitando(false);
    }, 900);
  };

  const handleKeyDown = (e) => {
    if (e.key === 'Enter') enviar();
  };

  return (
    <div className="chatbot">
      <header className="chatbot-header">
        <span className="dot"></span>
        <strong>IA Assistant</strong>
        <small>{digitando ? 'digitando...' : 'online'}</small>
      </header>

      <div className="mensagens">
        {mensagens.map(msg => (
          <Mensagem key={msg.id} {...msg} />
        ))}
        {digitando && <div className="digitando">● ● ●</div>}
        <div ref={fimRef} />   {/* âncora de scroll */}
      </div>

      <div className="input-area">
        <input
          value={input}
          onChange={e => setInput(e.target.value)}
          onKeyDown={handleKeyDown}
          placeholder="Mensagem..."
          disabled={digitando}
        />
        <button onClick={enviar} disabled={digitando}>Enviar</button>
      </div>
    </div>
  );
}

// Componente de mensagem separado
function Mensagem({ texto, isBot }) {
  return (
    <div className={`msg ${isBot ? 'msg--bot' : 'msg--user'}`}>
      {texto}
    </div>
  );
}
Conexão Semana 5: a função enviar() vai ser substituída por uma chamada real à API da OpenAI com fetch e async/await. O componente inteiro permanece igual — só a resposta do bot muda de estática para real.

Exercício 02

Crie um componente Contador com estado valor iniciado em 0. Adicione três botões: +1, -1 e Zerar. O contador nunca deve ficar abaixo de 0. Exiba o valor com cor verde se acima de 5, amarela se entre 1 e 5, cinza se 0.

Básico — useState + Condicional
import { useState } from 'react';

export default function Contador() {
  const [valor, setValor] = useState(0);

  const cor = valor > 5 ? '#4ade80' : valor > 0 ? '#fbbf24' : '#6b7280';

  return (
    <div>
      <p style={{ fontSize: '3rem', color: cor }}>{valor}</p>
      <button onClick={() => setValor(v => v + 1)}>+1</button>
      <button onClick={() => setValor(v => Math.max(0, v - 1))}>-1</button>
      <button onClick={() => setValor(0)}>Zerar</button>
    </div>
  );
}

Exercício 04

Crie um componente FormularioPrompt com: um textarea controlado para o prompt, um select para escolher o modelo (GPT-4o, Gemini, Llama), um contador de caracteres que muda de cor ao atingir 80% do limite de 500 chars, e um botão "Enviar" desabilitado quando o prompt estiver vazio.

Médio — múltiplos estados + condicional
import { useState } from 'react';

const LIMITE = 500;

export default function FormularioPrompt() {
  const [prompt,  setPrompt]  = useState('');
  const [modelo,  setModelo]  = useState('gpt-4o');
  const [enviado, setEnviado] = useState(false);

  const pct    = (prompt.length / LIMITE) * 100;
  const corCont = pct >= 100 ? '#f87171' : pct >= 80 ? '#fbbf24' : '#6b7280';

  const handleEnviar = () => {
    alert(`Enviando para ${modelo}: "${prompt}"`);
    setEnviado(true);
  };

  return (
    <form onSubmit={e => { e.preventDefault(); handleEnviar(); }}>
      <select value={modelo} onChange={e => setModelo(e.target.value)}>
        <option value="gpt-4o">GPT-4o</option>
        <option value="gemini">Gemini 1.5</option>
        <option value="llama">Llama 3.1</option>
      </select>
      <textarea
        value={prompt}
        onChange={e => { setPrompt(e.target.value); setEnviado(false); }}
        maxLength={LIMITE}
        rows={4}
        placeholder="Digite seu prompt..."
      />
      <p style={{ color: corCont }}>{prompt.length}/{LIMITE}</p>
      <button type="submit" disabled={!prompt.trim()}>
        {enviado ? '✓ Enviado' : 'Enviar'}
      </button>
    </form>
  );
}

Expanda o chatbot com: (a) um seletor de modo que muda entre "Formal" e "Casual" e altera as respostas estáticas do bot; (b) um botão "Limpar conversa" que reseta o estado para a mensagem inicial; (c) um contador de mensagens enviadas pelo usuário exibido no header. Desafio — Estado composto + props de callback
import { useState, useRef, useEffect } from 'react';

const RESPOSTAS = {
  formal:  ['Compreendido. Processando sua solicitação.', 'Obrigado pelo contato. Retorno em breve.'],
  casual:  ['Boa pergunta! 😄', 'Entendi, manda mais detalhes!', 'Interessante isso! 🤔'],
};

const MSG_INICIAL = [{ id: 0, texto: 'Olá! Como posso ajudar?', isBot: true }];

export default function ChatBotExpandido() {
  const [msgs,       setMsgs]       = useState(MSG_INICIAL);
  const [input,      setInput]      = useState('');
  const [digitando,  setDigitando]  = useState(false);
  const [modo,       setModo]       = useState('casual');
  const [enviadas,   setEnviadas]   = useState(0);
  const fimRef = useRef(null);

  useEffect(() => { fimRef.current?.scrollIntoView({ behavior: 'smooth' }); }, [msgs]);

  const enviar = () => {
    if (!input.trim() || digitando) return;
    setMsgs(p => [...p, { id: Date.now(), texto: input, isBot: false }]);
    setEnviadas(n => n + 1);
    setInput('');
    setDigitando(true);
    setTimeout(() => {
      const lista = RESPOSTAS[modo];
      setMsgs(p => [...p, {
        id: Date.now() + 1,
        texto: lista[Math.floor(Math.random() * lista.length)],
        isBot: true
      }]);
      setDigitando(false);
    }, 800);
  };

  const limpar = () => { setMsgs(MSG_INICIAL); setEnviadas(0); };

  return (
    <div>
      <header>
        <span>IA · {enviadas} msgs enviadas</span>
        <select value={modo} onChange={e => setModo(e.target.value)}>
          <option value="casual">Casual</option>
          <option value="formal">Formal</option>
        </select>
        <button onClick={limpar}>Limpar</button>
      </header>
      <div>
        {msgs.map(m => <div key={m.id}>{m.texto}</div>)}
        {digitando && <div>● ● ●</div>}
        <div ref={fimRef} />
      </div>
      <input value={input} onChange={e => setInput(e.target.value)}
             onKeyDown={e => e.key === 'Enter' && enviar()} />
      <button onClick={enviar} disabled={digitando}>Enviar</button>
    </div>
  );
}
