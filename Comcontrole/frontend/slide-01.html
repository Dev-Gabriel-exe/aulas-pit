HTML5 Semântico — estrutura que o navegador entende
Por que "semântico" importa?

HTML semântico é usar a tag que descreve o significado do conteúdo, não apenas o visual. Um <div> não diz nada; um <nav> diz "isso é navegação".

Para interfaces de IA isso vai importar muito: leitores de tela, indexadores, e os próprios LLMs que vão processar o DOM da sua página precisam entender a estrutura — não apenas ver texto.

Regra prática: antes de escrever <div>, pergunte se existe uma tag semântica para aquele conteúdo. Na maioria dos casos existe.
Estrutura base de um documento HTML5
HTML — esqueleto completo
<!DOCTYPE html>
<html lang="pt-BR">

<head>
  <meta charset="UTF-8"/>
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
  <title>Minha Página</title>
  <link rel="stylesheet" href="styles.css"/>
</head>

<body>

  <header>                       <!-- cabeçalho do site -->
    <nav>...</nav>               <!-- menu de navegação -->
  </header>

  <main>                         <!-- conteúdo principal (único por página) -->
    <section>                   <!-- seção temática com título -->
      <h2>Título da Seção</h2>
      <article>...</article>     <!-- conteúdo autônomo (post, card) -->
    </section>

    <aside>...</aside>           <!-- conteúdo secundário / sidebar -->
  </main>

  <footer>...</footer>         <!-- rodapé -->

  <script src="app.js"></script>  <!-- JS no final do body -->
</body>
</html>

Visualizando como o navegador enxerga essa estrutura:

<html>
<head> — metadados, título, CSS
<body>
<header> — logo, nav
<nav> — links de navegação
<main> — único por página!
<section> — seção temática
<article> — conteúdo autônomo
<aside> — sidebar / complementar
<footer> — rodapé

Relevância para IA: quando você usa Transformers.js ou TensorFlow.js para analisar conteúdo de uma página, a estrutura semântica determina o que o modelo consegue entender. Tags semânticas = contexto gratuito para o modelo.

Formulários — input types e validação nativa

Formulários são a principal forma de entrada de dados em interfaces de IA. Use o tipo correto de input e o navegador faz a validação de graça.

HTML — formulário semântico
<form action="/submit" method="POST" novalidate>

  <!-- label SEMPRE associado ao input via for/id -->
  <label for="nome">Nome completo</label>
  <input type="text" id="nome" name="nome"
         placeholder="Ana Lima" required autocomplete="name"/>

  <label for="email">E-mail</label>
  <input type="email" id="email" name="email" required/>

  <label for="prompt">Pergunta para a IA</label>
  <textarea id="prompt" name="prompt"
            rows="4" maxlength="500" required></textarea>

  <button type="submit">Enviar</button>

</form>
Tipos de input mais usados: text · email · password · number · range · checkbox · radio · file · date · search. Cada tipo ativa teclado virtual correto no mobile e validação nativa.

CSS3 Moderno — Flexbox, Grid, variáveis e responsividade
Variáveis CSS — o sistema de design que você controla

Variáveis CSS (custom properties) permitem criar um sistema de design consistente e mudar todo o visual com poucas linhas. São fundamentais em qualquer projeto real.

CSS — variáveis e uso
/* Definição no :root — disponível em todo o documento */
:root {
  --cor-primaria:    #f59e0b;
  --cor-fundo:       #0f172a;
  --cor-texto:       #e2e8f0;
  --cor-borda:       rgba(255,255,255,0.1);
  --raio:            12px;
  --sombra:          0 4px 20px rgba(0,0,0,0.4);
  --fonte-titulo:    'Montserrat', sans-serif;
}

/* Uso com var() */
.botao {
  background:   var(--cor-primaria);
  border-radius: var(--raio);
  box-shadow:   var(--sombra);
  color:        var(--cor-fundo);
}

/* Tema escuro em uma linha */
[data-tema="escuro"] {
  --cor-fundo: #020617;
  --cor-texto: #f8fafc;
}
Diferente de variáveis SASS/LESS, variáveis CSS são dinâmicas — podem ser alteradas com JavaScript em tempo real. Isso é muito útil para temas, modo escuro/claro e animações.
Flexbox — layout em uma dimensão

Flexbox organiza elementos em linha ou coluna. É a solução mais direta para centralizar, distribuir e alinhar itens dentro de um container.

CSS — Flexbox essencial
.container {
  display:         flex;
  flex-direction:  row;           /* row | column | row-reverse */
  justify-content: space-between; /* eixo principal */
  align-items:     center;        /* eixo cruzado */
  gap:             1rem;           /* espaço entre itens */
  flex-wrap:       wrap;           /* quebra de linha */
}

.item {
  flex: 1;          /* cresce para preencher o espaço */
  flex: 0 0 200px;  /* tamanho fixo de 200px */
  align-self: flex-end; /* alinhamento individual */
}

/* Centralizar qualquer coisa — a fórmula */
.centralizado {
  display:         flex;
  justify-content: center;
  align-items:     center;
}

CSS Grid — layout em duas dimensões

Grid controla linhas e colunas ao mesmo tempo. É a melhor ferramenta para layouts de página, dashboards e qualquer estrutura bidimensional.

CSS — Grid essencial
.grid-basico {
  display:               grid;
  grid-template-columns: 1fr 1fr 1fr;   /* 3 colunas iguais */
  gap:                   1rem;
}

/* Responsivo sem media query — o padrão profissional */
.grid-responsivo {
  display:               grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap:                   1.5rem;
}

/* Layout de página com áreas nomeadas */
.layout-pagina {
  display:             grid;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
  grid-template-columns: 260px 1fr;
  grid-template-rows:   auto 1fr auto;
  min-height:           100vh;
}

.layout-pagina header { grid-area: header; }
.layout-pagina aside  { grid-area: sidebar; }
.layout-pagina main   { grid-area: main; }
.layout-pagina footer { grid-area: footer; }

Design Responsivo — mobile-first

Mobile-first significa escrever o CSS para telas menores primeiro e adicionar complexidade para telas maiores. É o padrão da indústria — e faz seu CSS mais simples.

CSS — mobile-first com media queries
/* BASE — mobile (sem media query) */
.cards {
  display: grid;
  gap:     1rem;
  /* coluna única por padrão */
}

/* Tablet: >= 640px */
@media (min-width: 640px) {
  .cards {
    grid-template-columns: 1fr 1fr;
  }
}

/* Desktop: >= 1024px */
@media (min-width: 1024px) {
  .cards {
    grid-template-columns: repeat(3, 1fr);
  }
}

/* Breakpoints padrão do Tailwind (referência) */
/* sm: 640px | md: 768px | lg: 1024px | xl: 1280px */

/* clamp() — tipografia fluida sem media query */
h1 {
  font-size: clamp(1.5rem, 4vw, 3rem);
  /* mínimo: 1.5rem | preferido: 4vw | máximo: 3rem */
}
AbordagemComo funcionaQuando usar
Mobile-firstBase mobile, min-width para crescerSempre — padrão recomendado
Desktop-firstBase desktop, max-width para encolherProjetos legados apenas
auto-fit + minmaxGrid adapta sem media queryGalerias, cards, listas
clamp()Valor fluido com min e maxTipografia, espaçamentos

JavaScript ES6+ — a sintaxe moderna que o React exige
Arrow Functions — funções enxutas

Arrow functions não são apenas uma forma mais curta de escrever funções. Elas têm um comportamento diferente do this — o que importa muito em React e em callbacks de eventos.

JavaScript — Arrow functions
// Função tradicional
function dobrar(n) {
  return n * 2;
}

// Arrow function equivalente
const dobrar = (n) => n * 2;     // return implícito
const dobrar = n  => n * 2;     // parênteses opcionais com 1 param

// Com corpo de bloco (return obrigatório)
const processar = (texto) => {
  const limpo = texto.trim().toLowerCase();
  return limpo;
};

// Uso com métodos de array — o mais comum em React
const numeros = [1, 2, 3, 4, 5];

const dobrados  = numeros.map(n => n * 2);         // [2,4,6,8,10]
const pares     = numeros.filter(n => n % 2 === 0); // [2,4]
const soma       = numeros.reduce((acc, n) => acc + n, 0); // 15

// Renderizar lista em React (prévia da Semana 2)
const itens = ['HTML', 'CSS', 'JS'];
// itens.map(item => <li key={item}>{item}</li>)
Destructuring — extrair sem repetir

Destructuring é a sintaxe que permite extrair valores de objetos e arrays em uma linha. Em React, você vai usar isso a todo momento.

JavaScript — Destructuring
// ── Objetos ──
const usuario = { nome: 'Ana Lima', idade: 24, cidade: 'SP' };

// Sem destructuring
const nome   = usuario.nome;
const idade  = usuario.idade;

// Com destructuring
const { nome, idade } = usuario;

// Com renomeação e valor padrão
const { nome: nomeCompleto, pais = 'Brasil' } = usuario;

// Em parâmetros de função — muito comum em React
function Perfil({ nome, idade, cidade = 'Não informada' }) {
  return `${nome}, ${idade} anos, ${cidade}`;
}

// ── Arrays ──
const cores = ['vermelho', 'verde', 'azul'];
const [primeira, segunda] = cores;
const [, , terceira]       = cores;  // pular elementos

// useState retorna array — por isso se usa assim:
// const [contador, setContador] = useState(0);

// ── Spread e Rest ──
const { cidade, ...restante } = usuario; // rest: { nome, idade }
const novo = { ...usuario, cidade: 'RJ' }; // spread + override
Módulos ES6 — import / export

Módulos permitem dividir o código em arquivos e importar só o que você precisa. É o sistema de módulos que React, Next.js e todos os frameworks modernos usam.

JavaScript — Módulos
// ── utils.js — exportando ──
export function formatarPreco(valor) {
  return new Intl.NumberFormat('pt-BR', {
    style: 'currency', currency: 'BRL'
  }).format(valor);
}

export const API_URL = 'https://api.exemplo.com';

export default function saudacao(nome) {  // export default = 1 por arquivo
  return `Olá, ${nome}!`;
}

// ── app.js — importando ──
import saudacao from './utils.js';            // default
import { formatarPreco, API_URL } from './utils.js'; // named
import * as utils from './utils.js';        // tudo como namespace

// Importar de biblioteca externa
import { useState, useEffect } from 'react';
import axios from 'axios';
Template Literals e Async/Await

Template literals substituem concatenação:

JS — Template Literals
const nome = 'Ana';
const nota = 9.5;

// Concatenação (antigo)
"Olá, " + nome + "! Nota: " + nota;

// Template literal (moderno)
`Olá, ${nome}! Nota: ${nota}`

// Multiline — sem \n
const html = `
  <div class="card">
    <h2>${nome}</h2>
    <p>Nota: ${nota}</p>
  </div>
`;

Async/await — a forma limpa de lidar com Promises:

JS — Async/Await
// Buscar dados de uma API de IA
async function perguntarIA(prompt) {
  try {
    const resposta = await fetch('/api/ia', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ prompt })
    });

    if (!resposta.ok) throw new Error('Erro na API');

    const dados = await resposta.json();
    return dados.resposta;

  } catch (erro) {
    console.error(`Falha: ${erro.message}`);
    throw erro;
  }
}
Conexão com a Semana 4: esse padrão exato — async/await com fetch e tratamento de erro — é o que você vai usar para chamar APIs de IA (OpenAI, HuggingFace). Guarda esse template.
