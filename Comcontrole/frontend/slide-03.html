useEffect — efeitos colaterais e ciclo de vida
O que é um "efeito colateral"?

Em React, efeito colateral é qualquer coisa que acontece fora do fluxo puro de renderização: buscar dados de uma API, ouvir eventos, manipular o DOM diretamente, configurar timers ou sincronizar com sistemas externos.

useEffect é o hook que diz ao React: "depois de renderizar, execute isso".

Componente monta
→
useEffect roda
→
[] — só na montagem
Estado / prop muda
→
useEffect roda
→
[dep] — quando dep muda
Componente desmonta
→
cleanup roda
→
return () => ... — limpeza
useEffect — os três padrões essenciais
JSX — useEffect na prática
import { useState, useEffect } from 'react';

function ExemploEffect() {
  const [dados,   setDados]   = useState(null);
  const [query,   setQuery]   = useState('react');
  const [online,  setOnline]  = useState(true);

  // ── Padrão 1: [] — roda UMA VEZ ao montar ──────────────────────
  useEffect(() => {
    document.title = 'IA Assistant — Bem-vindo!';
    console.log('Componente montou');
  }, []); // array vazio = só na montagem

  // ── Padrão 2: [dep] — roda quando "query" muda ─────────────────
  useEffect(() => {
    if (!query) return;

    let cancelado = false; // evita atualizar estado após desmontar

    async function buscar() {
      const res  = await fetch(`https://api.exemplo.com/search?q=${query}`);
      const json = await res.json();
      if (!cancelado) setDados(json);
    }

    buscar();

    // cleanup — cancela se o componente desmontar antes da resposta
    return () => { cancelado = true; };
  }, [query]); // roda sempre que query mudar

  // ── Padrão 3: cleanup de event listener ────────────────────────
  useEffect(() => {
    const atualizar = () => setOnline(navigator.onLine);

    window.addEventListener('online',  atualizar);
    window.addEventListener('offline', atualizar);

    // ← cleanup: remove os listeners quando o componente desmontar
    return () => {
      window.removeEventListener('online',  atualizar);
      window.removeEventListener('offline', atualizar);
    };
  }, []); // sem deps = registra uma vez, limpa ao desmontar

  return <p>{online ? '🟢 Online' : '🔴 Offline'}</p>;
}
DependênciasQuando rodaCaso de uso típico
Sem arrayA cada renderRaramente — quase sempre um bug
[] vazioSó na montagemBusca inicial, event listeners, timers
[dep]Quando dep mudaBusca com filtro, sincronizar com prop
return () => {}Ao desmontar / antes de re-rodarRemover listeners, cancelar fetches
A regra das dependências: todo valor do componente que é usado dentro do useEffect deve estar no array de dependências. Se você usar query dentro do effect e não colocar [query], o efeito vai sempre enxergar o valor inicial — bug difícil de detectar.
useEffect para o chatbot — substituindo setTimeout

Na Semana 2 usamos setTimeout direto no handler. Com useEffect conseguimos reagir ao estado de forma mais limpa e com cleanup correto.

JSX — useEffect no chatbot
function ChatBot() {
  const [msgs,      setMsgs]      = useState([]);
  const [aguardando, setAguardando] = useState(false);
  const fimRef = useRef(null);

  // Scroll automático toda vez que mensagens mudam
  useEffect(() => {
    fimRef.current?.scrollIntoView({ behavior: 'smooth' });
  }, [msgs]);

  // Título da aba atualiza com número de mensagens
  useEffect(() => {
    document.title = msgs.length > 0
      ? `(${msgs.length}) IA Assistant`
      : 'IA Assistant';
  }, [msgs.length]);

  // Simula resposta da IA quando "aguardando" for true
  useEffect(() => {
    if (!aguardando) return;

    const timer = setTimeout(() => {
      setMsgs(p => [...p, {
        id: Date.now(), texto: 'Resposta da IA aqui!', isBot: true
      }]);
      setAguardando(false);
    }, 900);

    return () => clearTimeout(timer); // cleanup se desmontar
  }, [aguardando]);

  // ... resto do componente
}

Lifting state up e Context API — o estado que viaja
Lifting state up — quando filho precisa falar com pai

Props fluem para baixo. Para enviar dados de filho para pai, o pai passa uma função como prop. O filho chama essa função — isso é "lifting state up".

JSX — lifting state up
// ── O pai controla o estado ──────────────────────────────────────
function PainelIA() {
  const [modeloAtivo, setModeloAtivo] = useState('gpt-4o');
  const [historico,   setHistorico]   = useState([]);

  const adicionarMensagem = (msg) => {
    setHistorico(prev => [...prev, msg]);
  };

  return (
    <div>
      {/* Filho 1: seletor — ao mudar, notifica o pai */}
      <SeletorModelo
        modelo={modeloAtivo}
        onMudar={setModeloAtivo}    {/* função como prop ← */}
      />

      {/* Filho 2: chat — ao enviar, notifica o pai */}
      <ChatInput
        onEnviar={adicionarMensagem}  {/* função como prop ← */}
      />

      {/* Filho 3: lista — recebe o estado do pai */}
      <ListaMensagens msgs={historico} />
    </div>
  );
}

// ── O filho usa a função recebida como prop ──────────────────────
function SeletorModelo({ modelo, onMudar }) {
  return (
    <select value={modelo} onChange={e => onMudar(e.target.value)}>
      <option value="gpt-4o">GPT-4o</option>
      <option value="gemini">Gemini</option>
    </select>
  );
}
Regra prática: coloque o estado no ancestral comum mais próximo dos componentes que precisam dele. Se dois irmãos precisam do mesmo dado, sobe para o pai. Se muitos níveis precisam, é hora da Context API.
Context API — estado global sem prop drilling

Prop drilling é passar props por vários níveis intermediários só para chegar no componente que realmente precisa. Context resolve isso: você cria um "canal" de dados que qualquer componente da árvore pode acessar diretamente.

<TemaContext.Provider value={{ tema, setTema }}>
↓
App
↓
Layout
↓
Sidebar
↓
BotaoTema → useContext(TemaContext) ← acessa direto!
JSX — Context API passo a passo
// ── Passo 1: criar o contexto ── contexts/TemaContext.jsx ────────
import { createContext, useContext, useState } from 'react';

const TemaContext = createContext(null); // cria o canal

// Provider — envolve a árvore e fornece o valor
export function TemaProvider({ children }) {
  const [tema, setTema] = useState('escuro');

  const alternarTema = () =>
    setTema(t => t === 'escuro' ? 'claro' : 'escuro');

  return (
    <TemaContext.Provider value={{ tema, alternarTema }}>
      {children}
    </TemaContext.Provider>
  );
}

// Hook customizado — encapsula o useContext
export function useTema() {
  const ctx = useContext(TemaContext);
  if (!ctx) throw new Error('useTema deve ser usado dentro de TemaProvider');
  return ctx;
}

// ── Passo 2: envolver o app ── main.jsx ──────────────────────────
import { TemaProvider } from './contexts/TemaContext';

createRoot(document.getElementById('root')).render(
  <TemaProvider>
    <App />
  </TemaProvider>
);

// ── Passo 3: consumir em qualquer lugar ─ BotaoTema.jsx ──────────
import { useTema } from '../contexts/TemaContext';

function BotaoTema() {
  const { tema, alternarTema } = useTema(); // sem props!
  return (
    <button onClick={alternarTema}>
      {tema === 'escuro' ? '☀️ Claro' : '🌙 Escuro'}
    </button>
  );
}
Context não é substituto do useState. Use Context para dados verdadeiramente globais: tema, usuário logado, idioma, histórico de conversas. Para estado local de um componente, continue usando useState. Para estado complexo de app grande, considere Zustand (Semana 16).

React Router v6 — navegação client-side
O que é roteamento client-side?

Em apps tradicionais, cada URL faz uma nova requisição ao servidor. Com React Router, a navegação acontece no browser, sem recarregar a página. A URL muda, o componente muda, mas o estado global permanece.

/
→
<Home />
·
/chat
→
<Chat />
·
/chat/123
→
<Chat id="123"/>
·
/config
→
<Config />
Setup completo — BrowserRouter, Routes e Link
JSX — React Router v6 completo
// ── main.jsx ─────────────────────────────────────────────────────
import { BrowserRouter } from 'react-router-dom';

createRoot(document.getElementById('root')).render(
  <BrowserRouter>         {/* envolve tudo */}
    <TemaProvider>
      <App />
    </TemaProvider>
  </BrowserRouter>
);

// ── App.jsx — define as rotas ────────────────────────────────────
import { Routes, Route, Navigate } from 'react-router-dom';

export default function App() {
  return (
    <div>
      <Navbar />   {/* fora das rotas — sempre visível */}

      <Routes>
        <Route path="/"         element={<Home    />} />
        <Route path="/chat"     element={<Chat    />} />
        <Route path="/config"   element={<Config  />} />

        {/* Rota dinâmica — :id vira parâmetro */}
        <Route path="/chat/:id" element={<ChatDetalhe />} />

        {/* Rota 404 */}
        <Route path="*"         element={<Navigate to="/" />} />
      </Routes>
    </div>
  );
}

// ── Navbar.jsx — links de navegação ─────────────────────────────
import { Link, NavLink, useNavigate } from 'react-router-dom';

function Navbar() {
  const navegar = useNavigate(); // navegar por código

  return (
    <nav>
      {/* NavLink adiciona classe "active" na rota atual */}
      <NavLink to="/"       className={({ isActive }) => isActive ? 'ativo' : ''}>Home</NavLink>
      <NavLink to="/chat"    className={({ isActive }) => isActive ? 'ativo' : ''}>Chat</NavLink>
      <NavLink to="/config"  className={({ isActive }) => isActive ? 'ativo' : ''}>Config</NavLink>

      {/* Navegar por código — após login, por exemplo */}
      <button onClick={() => navegar('/chat')}>Ir para o Chat</button>
    </nav>
  );
}

// ── ChatDetalhe.jsx — ler parâmetro da URL ───────────────────────
import { useParams, useSearchParams } from 'react-router-dom';

function ChatDetalhe() {
  const { id } = useParams();  // /chat/42 → id = "42"

  // query string: /chat/42?modo=compacto → modo = "compacto"
  const [params] = useSearchParams();
  const modo = params.get('modo');

  return <h1>Chat #{id} — modo: {modo || 'padrão'}</h1>;
}
Link vs <a>: nunca use <a href="/chat"> em React Router — isso recarrega a página inteira, perdendo todo o estado. Sempre use <Link to="/chat"> ou useNavigate().
Rota protegida — redirecionar sem login

Um padrão muito comum em apps de IA: proteger rotas que exigem autenticação ou configuração (como ter uma chave de API definida).

JSX — rota protegida
import { Navigate, Outlet } from 'react-router-dom';

// Componente de rota protegida genérico
function RotaProtegida({ permitido }) {
  return permitido
    ? <Outlet />                    // renderiza a rota filha
    : <Navigate to="/config" replace />; // redireciona
}

// Usar no App.jsx
<Routes>
  <Route path="/"       element={<Home />} />
  <Route path="/config" element={<Config />} />

  {/* Rotas que exigem API key configurada */}
  <Route element={<RotaProtegida permitido={!!apiKey} />}>
    <Route path="/chat"      element={<Chat />} />
    <Route path="/dashboard" element={<Dashboard />} />
  </Route>
</Routes>
Conexão Semana 5: na Semana 5 você vai proteger a rota /chat verificando se o usuário configurou uma chave de API. Sem chave → redireciona para /config. O padrão acima é exatamente o que você vai usar.

O que vimos na Semana 3
useEffect

createContext → Provider com value → useContext (ou hook customizado). Para tema, usuário, idioma e dados globais.
